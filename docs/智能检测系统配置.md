# 🤖 YYC³ 自动智能检测系统

## 📋 系统概述

YYC³ 组件库已配置完整的 Git
Hooks 自动检测系统，在每次提交和推送前自动运行智能检测。

---

## ✅ 已激活的自动检测

### 1. **Pre-Commit Hook** (提交前自动检测)

**触发时机**: `git commit` 命令执行前

**检测内容**:

- 📦 **文件大小检查** - 检测大文件（> 1MB）
- 🔐 **敏感信息检查** - 检测密码、API Key、私钥等
- 🎨 **代码格式检查** - Prettier 格式验证
- 🔍 **语法检查** - ESLint 快速检查
- 📝 **类型检查** - TypeScript 类型验证

**配置文件**: `.git/hooks/pre-commit`

---

### 2. **Pre-Push Hook** (推送前自动检测)

**触发时机**: `git push` 命令执行前

**检测内容**:

- ✅ 所有 Pre-Commit 检查项
- 🧪 **单元测试** - 运行所有测试套件
- 🏗️ **构建检查** - 验证项目构建
- 🔒 **依赖安全** - 检查依赖漏洞
- ⚡ **性能测试** - 可选的性能基准测试

**配置文件**: `.git/hooks/pre-push`

---

### 3. **Commit-Msg Hook** (提交信息检查)

**触发时机**: `git commit` 信息输入后

**检测内容**:

- 📝 **提交格式检查** - 验证提交信息格式

**正确格式**: `<type>(<scope>): <subject>`

**支持的类型**:

```
feat     - 新功能
fix      - 修复 bug
docs     - 文档更改
style    - 代码格式
refactor - 重构
perf     - 性能优化
test     - 增加测试
build    - 构建变动
ci       - CI 配置变动
chore    - 其他更改
revert   - 回退
```

**示例**:

```bash
feat(ui): 添加新的 Button 组件
fix(hooks): 修复 useChannelConfig 的 bug
docs(readme): 更新安装说明
```

---

## 🚀 使用方式

### 正常提交（自动运行检测）

```bash
# 提交代码
git add .
git commit -m "feat(ui): 添加新的 Button 组件"
# ↑ 会自动运行 pre-commit 检查

# 推送代码
git push
# ↑ 会自动运行 pre-push 检查
```

---

### 跳过检测（不推荐）

```bash
# 跳过提交前检查
git commit --no-verify -m "feat: 临时提交"

# 跳过推送前检查
git push --no-verify
```

⚠️ **注意**: 仅在明确知道后果时使用 `--no-verify`

## 🎯 最佳实践

### 1. 提交前检查失败

```bash
# 1. 查看错误信息
# 2. 运行快速修复
pnpm format

# 3. 重新提交
git add .
git commit -m "feat: 新功能"
```

---

### 2. 推送前检查失败

```bash
# 1. 查看详细报告
cat .codebuddy/check-reports/check-*.json

# 2. 修复问题
pnpm lint          # 修复 ESLint 错误
pnpm typecheck     # 检查类型错误
pnpm test:run      # 运行测试

# 3. 重新推送
git push
```

---

### 3. 敏感信息检测

如果检测到敏感信息：

```bash
# 1. 移除敏感信息
# 2. 使用环境变量
# 3. 添加到 .gitignore
echo ".env" >> .gitignore

# 4. 重新提交
git add .
git commit -m "fix: 移除敏感信息"
```

## 📞 常见问题

### Q: 如何临时禁用某个检查？

**A**: 不推荐禁用，但可以使用 `--no-verify` 跳过所有检查

---

### Q: 检测太慢怎么办？

**A**: 使用快速检查模式：

```bash
pnpm check:quick  # 仅检查变更文件
```

---

### Q: 如何自定义检测规则？

**A**: 编辑对应的配置文件：

- ESLint: `.eslintrc.js`
- Prettier: `.prettierrc`
- TypeScript: `tsconfig.json`
