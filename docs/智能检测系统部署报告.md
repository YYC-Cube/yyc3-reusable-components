## 📊 检测流程

### 提交流程

```
git commit
    ↓
Pre-Commit Hook
    ├─ 文件大小检查
    ├─ 敏感信息检测
    ├─ 快速代码检查
    │   ├─ ESLint
    │   ├─ TypeScript
    │   └─ Prettier
    └─ 提交信息格式检查
    ↓
✅ 提交成功 / ❌ 提交失败
```

```
git push

    ├─ 安全检查
    └─ 性能测试（可选）

✅ 推送成功 / ❌ 推送失败
```

## 🎯 使用指南

### 日常开发流程

1. **开发新功能**

   ```bash
   # 创建功能分支
   node scripts/branch-manager.js create feature/123-new-ui --base develop

   # 开发过程中定期检查
   pnpm check:quick

   # 提交代码
   git add .
   git commit -m "feat(ui): 添加新组件"
   ```

2. **推送代码**

   ```bash
   # 推送前完整检查
   pnpm check:full

   # 推送到远程
   git push origin feature/123-new-ui
   ```

3. **合并到主分支**

   ```bash
   # 切换到目标分支
   git checkout develop

   # 合并
   git merge --no-ff feature/123-new-ui

   # 推送
   git push origin develop
   ```

### 快速命令

```bash
# 检查代码
pnpm check

# 查看分支
pnpm branch:analyze

# 初始化系统
node scripts/setup-smart-check.js

# 安装 Git Hooks
pnpm setup:hooks
```

## ⚠️ 注意事项

### 1. 环境要求

- ✅ Node.js >= 18.0.0
- ✅ pnpm >= 8.0.0
- ✅ Git >= 2.0.0

### 2. 首次使用

首次运行可能需要较长时间，系统会：

- 安装所有依赖
- 初始化 Git Hooks
- 构建缓存
- 运行完整检测

### 3. 跳过检查

虽然可以跳过检查，但**强烈不推荐**：

```bash
git commit --no-verify
git push --no-verify
```

### 4. 分支保护

受保护的分支需要：

- 通过所有检查
- 通过 PR 审核
- 满足审核者数量要求
