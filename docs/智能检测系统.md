# YYC³ 智能代码检测系统

> 企业级代码质量检测和管理系统，确保代码质量和团队协作效率

## 📋 概述

YYC³ 智能代码检测系统是一个全面的代码质量管理解决方案，集成了语法检查、类型检查、测试运行、代码格式化、安全检查等多种检测功能，通过 Git
Hooks 和 CI/CD 流程自动执行，确保代码质量始终保持在最高标准。

## ✨ 核心功能

### 1. 智能代码检测 (`smart-check.js`)

全面的代码质量检测工具，支持多种检测模式：

- **ESLint 代码质量检查** - 语法、代码规范、最佳实践
- **TypeScript 类型检查** - 静态类型分析
- **Prettier 代码格式化** - 统一代码风格
- **单元测试运行** - 自动运行测试套件
- **构建检查** - 确保项目可正常构建
- **依赖安全检查** - 检测已知安全漏洞
- **性能测试** - 可选的性能基准测试

**检测模式**：

```bash
# 标准检查（推荐）
pnpm check

# 快速检查（跳过测试）
pnpm check:quick

# 完整检查（包含所有测试）
pnpm check:full
```

### 2. Git Hooks 自动检测

#### Pre-Commit Hook

提交前快速检查，确保每次提交的代码质量：

- 文件大小检查
- 敏感信息检测
- 快速代码检查（语法、类型、格式）
- 提交信息格式验证

**使用方式**：

```bash
git commit  # 自动运行 pre-commit 检查
git commit --no-verify  # 跳过检查（不推荐）
```

#### Pre-Push Hook

推送前完整检查，确保推送的代码质量：

- 分支保护检查
- 冲突检测
- 完整代码检查
- 测试运行
- 构建验证

**使用方式**：

```bash
git push  # 自动运行 pre-push 检查
git push --no-verify  # 跳过检查（不推荐）
```

### 3. 分支管理工具 (`branch-manager.js`)

强大的分支管理工具，支持分支策略：

**支持的分支类型**：

| 分支类型    | 说明                            | 受保护 |
| ----------- | ------------------------------- | ------ |
| `main`      | 主分支 - 生产环境代码           | ✓      |
| `develop`   | 开发分支 - 集成测试环境         | ✓      |
| `staging`   | 预发布分支 - 预发布环境         | ✓      |
| `feature/*` | 功能分支 - 新功能开发           | ✗      |
| `bugfix/*`  | 修复分支 - Bug修复              | ✗      |
| `hotfix/*`  | 紧急修复分支 - 生产环境紧急修复 | ✗      |
| `release/*` | 发布分支 - 版本发布准备         | ✓      |

**命令示例**：

```bash
# 查看所有分支
pnpm branch:list

# 分析分支状态
pnpm branch:analyze

# 创建新分支
node scripts/branch-manager.js create feature/123-new-ui --base develop

# 删除分支
node scripts/branch-manager.js delete feature/old-feature

# 合并分支
node scripts/branch-manager.js merge feature/123-new-ui

# 生成分支报告
pnpm branch:report
```

### 4. CI/CD 集成

自动化的持续集成流程，确保代码质量：

**工作流程**：

1. **快速检查** - 对所有推送运行
2. **类型检查** - TypeScript 静态分析
3. **代码质量检查** - ESLint + Prettier
4. **单元测试** - 运行测试套件
5. **构建检查** - 验证构建成功
6. **安全检查** - 依赖漏洞扫描
7. **性能测试** - 可选的性能基准
8. **报告生成** - 生成检测报告

**查看工作流**：`.github/workflows/smart-check.yml`

## 🚀 快速开始

### 1. 安装和初始化

```bash
# 运行初始化脚本
node scripts/setup-smart-check.js
```

初始化脚本会自动完成：

- ✓ 环境检查（Node.js、pnpm、Git）
- ✓ 创建必要的目录结构
- ✓ 安装 Git Hooks
- ✓ 安装项目依赖
- ✓ 运行测试检查

### 2. 手动运行检查

```bash
# 标准检查（推荐日常使用）
pnpm check

# 快速检查（提交前快速验证）
pnpm check:quick

# 完整检查（推送前完整验证）
pnpm check:full
```

### 3. Git 工作流

```bash
# 正常提交流程
git add .
git commit -m "feat(ui): 添加新组件"
# 自动运行 pre-commit 检查

# 推送到远程
git push origin feature/123-new-ui
# 自动运行 pre-push 检查
```

### 4. 分支管理

```bash
# 查看分支状态
pnpm branch:analyze

# 创建功能分支
node scripts/branch-manager.js create feature/123-new-ui

# 查看分支列表
pnpm branch:list
```

## 📊 检测报告

系统会自动生成详细的检测报告，保存在 `.codebuddy/` 目录下：

- `check-reports/` - 代码检测报告
- `branch-reports/` - 分支状态报告
- `performance-reports/` - 性能测试报告

**报告内容**：

- 检测时间戳
- 检测结果统计
- 详细检查项结果
- Git 信息（分支、提交、变更文件）
- 成功率计算

## ⚙️ 配置和自定义

### 跳过检查

虽然可以跳过检查，但不推荐：

```bash
# 跳过 pre-commit
git commit --no-verify -m "message"

# 跳过 pre-push
git push --no-verify
```

### 自定义检查规则

可以修改以下文件自定义检查：

- `smart-check.js` - 调整检测流程和阈值
- `pre-commit-check.js` - 自定义提交前检查
- `pre-push-check.js` - 自定义推送前检查
- `branch-manager.js` - 自定义分支策略

### 环境变量

```bash
# CI 环境标识
CI=true

# Snyk 安全检查令牌
SNYK_TOKEN=your_token_here
```

## 🎯 最佳实践

### 1. 提交代码前

```bash
# 1. 运行快速检查
pnpm check:quick

# 2. 修复发现的问题

# 3. 提交代码
git add .
git commit -m "feat(ui): 添加新组件"
```

### 2. 推送代码前

```bash
# 1. 运行完整检查
pnpm check:full

# 2. 确保所有测试通过

# 3. 推送到远程
git push origin feature/123-new-ui
```

### 3. 合并到主分支

```bash

git checkout develop

# 2. 拉取最新代码
git pull origin develop

# 3. 合并功能分支
git merge --no-ff feature/123-new-ui

# 4. 推送到远程
git push origin develop
```

## 🔍 故障排查

### 问题：pre-commit 失败

**解决方案**：

1. 查看错误信息
2. 运行 `pnpm check:quick` 获取详细报告
3. 修复问题
4. 重新提交

### 问题：pre-push 失败

**解决方案**：

1. 查看错误信息
2. 运行 `pnpm check:full` 获取详细报告
3. 修复问题
4. 重新推送

### 问题：检测脚本异常

**解决方案**：

1. 检查 Node.js 版本（需要 >= 18）
2. 检查 pnpm 版本
3. 运行 `pnpm install` 重新安装依赖
4. 运行 `node scripts/setup-smart-check.js` 重新初始化

## 📈 性能优化

### 利用缓存

系统会自动利用 Turbo 缓存加速检查：

- 首次运行：完整检查（较慢）
- 后续运行：增量检查（快速）

### 并行执行

CI/CD 流程采用并行执行策略：

- 类型检查、代码检查、测试同时运行
- 显著减少总检测时间

## 🤝 团队协作

### 分支保护规则

- `main` 分支：需要 2 个审核者，必须通过所有检查
- `develop` 分支：需要 1 个审核者，必须通过所有检查
- `staging` 分支：需要 2 个审核者，必须通过所有检查

### 提交信息规范

使用约定式提交：

```
<type>(<scope>): <subject>

type:
  feat:     新功能
  fix:      修复bug
  docs:     文档更改
  style:    代码格式
  refactor: 重构
  perf:     性能优化
  test:     测试相关
  build:    构建相关
  ci:       CI配置
  chore:    其他更改
  revert:   回退
```
