## 🎯 问题分析

### 为什么有这么多分支？

1. **Dependabot 自动创建**
   - 项目启用了 Dependabot 自动依赖更新
   - 每个依赖更新都会创建一个新分支和 PR

2. **PR 未处理**
   - 12个 PR 处于打开状态，等待审核和合并
   - 1个 PR 已合并但分支未删除

3. **缺少自动清理**
   - 合并后的分支没有自动删除
   - 关闭的 PR 分支没有被清理

### 潜在影响

**正面影响**:

- ✅ 保持依赖更新
- ✅ 提高安全性
- ✅ 自动化依赖管理

**负面影响**:

- ⚠️ 分支列表混乱
- ⚠️ 可能影响 CI/CD 性能
- ⚠️ 增加维护负担

### 方案二：优化 Dependabot 配置

修改 `.github/dependabot.yml`：

```yaml
version: 2
registries:
  npm-github:
    type: npm-registry
    url: https://npm.pkg.github.com
    username: ${{ secrets.DEPENDABOT_USERNAME }}
    password: ${{ secrets.DEPENDABOT_TOKEN }}

updates:
  # GitHub Actions 更新
  - package-ecosystem: 'github-actions'
    directory: '/'
    schedule:
      interval: 'weekly'
      day: 'monday'
      time: '09:00'
    open-pull-requests-limit: 3
    labels:
      - 'dependencies'
      - 'github-actions'
    commit-message:
      prefix: 'ci'
      include: 'scope'
    # 自动合并次要更新
    versioning-strategy: increase

  # npm 依赖更新
  - package-ecosystem: 'npm'
    directory: '/'
    schedule:
      interval: 'weekly'
      day: 'monday'
      time: '09:00'
    open-pull-requests-limit: 5
    labels:
      - 'dependencies'
      - 'npm'
    commit-message:
      prefix: 'chore'
      include: 'scope'
    # 分组更新
    groups:
      production-dependencies:
        dependency-type: 'production'
        update-types:
          - 'minor'
          - 'patch'
      development-dependencies:
        dependency-type: 'development'
        update-types:
          - 'minor'
          - 'patch'
    # 忽略特定版本
    ignore:
      - dependency-name: 'typescript'
        versions: ['6.x']
      - dependency-name: '@types/node'
        versions: ['25.x']
```

**优化要点**:

- 限制同时打开的 PR 数量
- 分组依赖更新
- 设置更新时间表
- 自动合并次要更新

### 方案三：启用自动合并

创建 `.github/workflows/auto-merge.yml`：

```yaml
name: Auto Merge Dependabot PRs

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  auto-merge:
    runs-on: ubuntu-latest
    if: github.actor == 'dependabot[bot]'
    steps:
      - name: Check PR
        uses: actions/github-script@v8
        with:
          script: |
            const pr = context.payload.pull_request;

            // 只自动合并次要和补丁更新
            if (pr.title.includes('minor') || pr.title.includes('patch')) {
              // 等待 CI 通过
              await github.rest.pulls.merge({
                owner: context.repo.owner,
                repo: context.repo.repo,
                pull_number: pr.number,
                merge_method: 'squash'
              });
            }
```

### 方案四：批量处理 PR

使用 GitHub CLI 批量处理：

```bash
# 查看所有 Dependabot PR
gh pr list --author app/dependabot --state open

# 批量合并测试通过的 PR
gh pr list --author app/dependabot --state open --json number,title --jq '.[] | select(.title | contains("patch")) | .number' | xargs -I {} gh pr merge {} --squash --delete-branch

# 批量关闭不需要的 PR
gh pr list --author app/dependabot --state open --json number,title --jq '.[] | select(.title | contains("major")) | .number' | xargs -I {} gh pr close {} --delete-branch
```

## ⚠️ 注意事项

1. **备份重要更改**
   - 在删除分支前，确保重要更改已合并
   - 检查是否有未合并的重要功能

2. **测试依赖更新**
   - 合并前运行完整测试
   - 检查是否有破坏性变更
   - 验证兼容性

3. **团队协作**
   - 通知团队成员分支清理计划
   - 确认没有人正在使用这些分支
   - 记录清理决策

4. **回滚计划**
   - 如果删除错误，可以从远程恢复
   - 保留删除记录和日志
